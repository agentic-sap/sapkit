# HCM - Human Capital Management Development Workflows
# HCM - 인적 자본 관리 개발 워크플로우

## Workflow 1: Employee Hire Action via BAPI
### Steps
1. Enqueue employee record: BAPI_EMPLOYEE_ENQUEUE for personnel number (or new number from number range)
2. Create IT0000 (Actions): BAPI_PERSDATA_CREATE with action type = '01' (Hiring), reason, entry date
3. Create IT0001 (Org Assignment): company code, personnel area, employee group/subgroup, org unit, position
4. Create IT0002 (Personal Data): name, gender, date of birth, nationality
5. Create IT0006 (Addresses): home address
7. Create IT0007 (Planned Working Time): work schedule rule
8. Create IT0008 (Basic Pay): wage type, amount, pay scale
9. Dequeue: BAPI_EMPLOYEE_DEQUEUE; commit
10. Verify in PA10 (Personnel File)

### Required MCP Tools
- `GetFunctionModule` — read BAPI_PERSDATA_CREATE interface for each infotype
- `GetTable` — inspect PA0001, PA0002, PA0008 structures
- `CreateProgram` — scaffold hire action interface

### Related Config
- Personnel Actions: OOPA / V_T529A
- Employee Groups: V_T501
- Pay Scale Structure: V_T510

## Workflow 2: Custom Payroll Wage Type Calculation (Schema Rule)
### Steps
1. Identify payroll schema: PE01 to view schema (e.g., X000 for international, ZXYZ for custom)
2. Create custom calculation rule in PE02 (Z-namespace): define wage type derivation logic
3. In rule: use operations ADDWT (add wage type), MULTI (multiply), LIMIT (apply limits)
4. Insert rule call into schema at correct processing point via COPY of relevant schema section
5. Create Z-wage type: define in V_512W with processing class 01 (Relevant to payroll) and evaluation class
6. Test: run payroll simulation via BAPI_PAYROLL_SIMULATION or PC00_M99_CALC in test mode
7. Read results: PYXX_READ_PAYROLL_RESULT and verify RT table contains new wage type

### Required MCP Tools
- `GetProgram` — read payroll schema program generated by PE01
- `GetView` — inspect V_512W for wage type definitions
- `GetTable` — inspect RT, CRT payroll result structures

### Related Config
- Payroll Schema: PE01
- Calculation Rules: PE02
- Wage Types: OH11 / V_512W

## Workflow 3: Absence Quota Accrual Automation
### Steps
1. Configure quota type in V_T556 (Leave Types) with accrual settings
2. Implement BAdI PT_ARQ00_QUOTA_ACCOUNT for custom accrual calculation
3. In BAdI: read employee data (IT0001, IT0007) to determine entitlement based on seniority/grade
4. Calculate accrual amount: e.g., 1.5 days per month for employees > 3 years
5. Update IT2006 (Absence Quotas): BAPI_ABSENCE_CREATE or FM HR_TIME_QUOTA_TRANS_CREATE
6. Schedule accrual: background job running program RPTQTA00 monthly
7. Monitor quotas via PT63 (Time Account Overview)

### Required MCP Tools
- `GetClass` — inspect BAdI PT_ARQ00_QUOTA_ACCOUNT interface
- `CreateClass` — implement quota accrual calculation
- `GetTable` — inspect PA2006 (absence quota), T556 (quota types)
- `GetFunctionModule` — read HR_TIME_QUOTA_TRANS_CREATE

### Related Config
- Absence Types: V_T554S
- Work Schedule Rules: V_T508A
- Time Evaluation Schema: V_T52C5
